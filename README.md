# Spiking-Bug — лаборатория импульсного агента

Исследовательский стенд: 2D-агент-жук с импульсной нейронной сетью (LIF), управляемой через визуальный редактор в браузере.

* **Бэк:** Python + NumPy (векторизованная LIF-сеть).
* **Фронт:** vanilla JS + Canvas; Socket.IO для живой связи.
* **Размер сети:** до 512 нейронов, плотная матрица весов, шаг сети — десятки микросекунд.
* **Цель:** руками собрать схему из 10–100 нейронов, увидеть как агент ведёт себя в среде и как сигналы пробегают по синапсам.

## Как работает сеть

### LIF — модель нейрона

Каждый нейрон — «дырявая копилка»: в неё капает входной ток, она помаленьку утекает, и когда уровень переполняется — нейрон стреляет (спайк) и обнуляется. На каждом шаге `dt = 10 мс` для всех нейронов сразу:

```
V ← V·(1 − leak) + I_синапс + I_внешн + шум
если V ≥ threshold:  спайк, V ← v_reset, входит в рефрактерный период
```

Вся математика — один matvec в NumPy ([snn.py](snn.py)).

### Параметры нейрона

| Параметр | Что делает | Типично |
|---|---|---|
| **threshold** | Уровень, при котором нейрон стреляет | 1.0 |
| **leak** | Доля `V`, утекающая за такт. `0.02` — медленная утечка (сенсоры), `0.12` — быстрая (моторы). `0` — идеальный интегратор | 0.02 – 0.2 |
| **v_reset** | До чего сбрасывается `V` после спайка | 0 |
| **refractory** | Сколько тактов нейрон «глух» после спайка. Ограничивает максимальную частоту: `1 / ((refr+1)·dt)`. При `refr=2` → ~33 Hz max | 1 – 5 |
| **noise σ** | Стандартное отклонение гауссовского шума, добавляемого к `V`. По умолчанию 0; для inter-нейронов можно поставить ~0.05 чтобы дать «фоновую» активность | 0 – 0.1 |

### Параметры синапса

| Параметр | Что делает |
|---|---|
| **weight** | Когда нейрон-источник стреляет, к мембране получателя добавляется `weight`. **Положительный** = возбуждение (синяя стрелка), **отрицательный** = торможение (красная). При \|weight\| ≥ threshold один спайк гарантированно вызывает спайк-получатель |
| **delay** | Через сколько тактов спайк дойдёт от источника до получателя. На `dt=10мс`: delay=1 → 10мс, delay=5 → 50мс. На канвасе видно бегущую точку — это летящий импульс по аксону |

### F-I кривая сенсоров

При параметрах `leak=0.02, threshold=1, refr=2`:

| Входной ток I | Частота спайков |
|---|---|
| 0.020 | 0 Hz (не дотягивает) |
| 0.025 | ~1 Hz |
| 0.10 | ~7 Hz |
| 0.50 | ~20 Hz |
| 1.00 | ~33 Hz (максимум) |

Сенсоры берут «сырой» сигнал среды (например, сумму `K/d²` от всех еды в секторе) и нормируют в `[0, 1]` относительно «эталона насыщения», получая ток `I`. Дальше — обычная LIF-динамика.

## Среда и агент

### Арена

Прямоугольное поле фиксированного размера (по умолчанию **1100×700 px**), не меняется при ресайзе окна. Стенки непроходимы. В арене могут быть:

* **Еда** (зелёные кружочки) — даёт +0.1 к hp, обнуляет голод. Поддерживается фиксированное количество (ползунок «еды в среде»): когда одну съели, спавнится новая. Слайдер в 0 — спавна нет.
* **Угрозы** (красные шары, пульсируют) — наносят 0.4 hp/с при касании. Тоже фиксированное количество. У каждой свой TTL (ползунок «жизнь угрозы»): когда время вышло, угроза исчезает и в новом месте появляется свежая. Слайдер в 0 — все угрозы немедленно убираются.
* **Препятствия** — статичные серые прямоугольники, ставятся drag-ом.

### Агент-жук

* **Тело**: эллипс ~9 px радиус.
* **Движение**: каждый спайк `motor_forward` сдвигает на 1.6 px, `motor_backward` — на −1.1 px, `motor_left/right` — поворот на ~4.6° (рассчитан так что 33 Hz моторных спайков = около 50 px/с или 150°/с поворота).
* **Лидары**: 5 фронтальных лучей в веере ±50°. Каждый сообщает дистанцию до ближайшего препятствия (или стены) в радиусе 110 px. Ток: `I = 1 − dist/range` (близко → сильный сигнал).
* **Сенсоры еды и угрозы**: с **угловыми секторами чувствительности** относительно «носа» агента:
    | Сенсор | Сектор |
    |---|---|
    | `sensor_food_left` | −120° … +2° (слева сзади до узкой фронтальной зоны) |
    | `sensor_food_right` | −2° … +120° |
    | `sensor_threat_left` | −150° … 0° |
    | `sensor_threat_right` | 0° … +150° |

    Сектора визуализируются полупрозрачными «веерами» вокруг агента. У еды конусы пересекаются только в узком фронтальном клине **4°** (±2° от направления взгляда) — стерео работает почти исключительно строго прямо по курсу, по бокам сенсоры однозначно «левый/правый». Сектора угроз шире и встречаются в одной точке — ровно по 12 часов.
* **Голод**: `hunger ∈ [0, 1]`, растёт со скоростью `hunger_rate` (по умолчанию 0.05/с — полный голод за 20 с). При поедании еды → 0. `sensor_hunger` получает ток = `hunger` напрямую.
* **Усталость**: `fatigue ∈ [0, 1]`, прибавляется `fatigue_action_gain` за каждый моторный спайк, спадает `fatigue_decay`/с в покое. `sensor_fatigue` получает ток = `fatigue`.

### Дефолтные нейроны

Системные, удалить нельзя:

| ID | Тип | Назначение |
|---|---|---|
| `sensor_food_left` / `sensor_food_right` | sensor | Левый/правый сенсор еды (с сектором) |
| `sensor_threat_left` / `sensor_threat_right` | sensor | Левый/правый сенсор угрозы (с сектором) |
| `sensor_hunger` | sensor | Уровень голода |
| `sensor_fatigue` | sensor | Уровень усталости |
| `lidar_0` … `lidar_4` | sensor | 5 фронтальных лидаров (слева → справа) |
| `motor_forward` / `motor_backward` | motor | Движение вперёд/назад |
| `motor_left` / `motor_right` | motor | Поворот на месте |

## Интерфейс

### Верхняя панель

* **▶ Старт / ⏸ Пауза** — управление симуляцией.
* **⟲ Сбросить агента** — телепортирует в центр, обнуляет hp/голод/усталость.
* **⟲ Сбросить сеть** — обнуляет мембраны и спайк-счётчики (топология остаётся).
* **Частота сети** (1–100 Hz, дефолт 100) — slow-motion: на 10 Hz всё (агент, импульсы, голод) идёт в 10× медленнее реального времени, но динамика самой сети не искажается. Удобно чтобы наблюдать как импульсы пробегают по синапсам.
* **Статистика**: съеденная еда, hp, голод, усталость, симуляционное время.

### Среда (левая панель)

**Инструменты** (кнопки-переключатели):
* **↖ Курсор** — выбор объекта (для просмотра).
* **🟢 Еда** / **🔴 Угроза** — клик ставит объект.
* **▣ Препятствие** — drag (от угла к углу).
* **🐞 Агент** — клик телепортирует жука.
* **✕ Стереть** — клик по объекту удаляет его.

**Ползунки**:
* **еды в среде** (0–30) — целевое количество. 0 = спавна нет, существующая еда сохраняется.
* **угроз в среде** (0–15) — целевое количество. 0 = все угрозы немедленно исчезают.
* **жизнь угрозы, с** (2–60) — TTL каждой угрозы; по истечении она заменяется новой в другом месте.
* **голод/с** — скорость прироста голода.
* **усталость/действ.** — прибавка за каждый моторный спайк.
* **отдых/с** — скорость спадания усталости.

**Очистка**: три кнопки «очистить еду / угрозы / препятствия».

**Управление вьюпортом**:
* Колесо мыши = зум.
* Shift + drag = панорамирование.
* ⊙ 1:1 = сброс зума.

**Ручное управление мотонейронами** (для отладки): кнопки внизу или клавиши **W A S D** — пока зажата, в соответствующий мотор впрыскивается постоянный ток (имитирует уверенный сигнал).

**HUD** (левый верх): полоски hp (зелёная), голода (оранжевая), усталости (фиолетовая).

### Редактор сети (правая панель)

**Инструменты**:
* **↖ Курсор** —
    * клик по нейрону → выделить и редактировать в правой колонке;
    * клик и drag по нейрону → переместить;
    * **shift + click** по нейрону → добавить/убрать из множественного выделения;
    * клик по пустому месту + drag → **рамка выделения**; всё что попало внутрь, выделяется как группа;
    * drag любого выделенного нейрона → двигает все выделенные;
    * **Delete / Backspace** удаляет всё выделенное (системные пропускаются);
    * клик по стрелке-синапсу → выделить синапс.
* **+ Нейрон** — клик в пустом месте создаёт inter-нейрон с дефолтными параметрами.
* **→ Синапс** — клик по источнику, потом по цели. Создаёт синапс с весом `1.0` и задержкой `5` (без диалогов). Параметры правишь в свойствах.
* **⬚ Рамка** — drag в пустом месте создаёт прямоугольник-«группу» позади нейронов. У неё есть цвет, надпись и **комментарий** (виден как tooltip при наведении на надпись — рядом с надписью появляется значок ⓘ если коммент есть).
* **✕ Удалить** — клик по нейрону / синапсу / рамке удаляет.

**Свойства** (правая колонка):
* Для **нейрона**: имя, порог, утечка, V_reset, рефрактерный период, шум σ. Кнопка «Применить».
* Для **синапса**: вес, задержка. Кнопка «Применить».
* Для **рамки**: текст, цвет (color picker), коммент (textarea).
* Для **множественного выделения**: одна кнопка «удалить выделение».

**Визуализация активности**:
* **Цвет нейрона**: голубой = sensor, оранжевый = motor, зелёный = inter.
* **Яркость**: пропорциональна текущему `V/threshold`. Тёмный — далеко от порога, яркий — почти выстрелит.
* **Жёлтый ореол** — только что был спайк (вспышка 250 мс).
* **Стрелка синапса**: толщина = |weight|; синяя = +, красная = −.
* **Бегущая точка** на стрелке — летящий импульс по аксону. Жёлтая если несёт возбуждение, красная — торможение. Скорость соответствует `delay · dt` в симуляционном времени.

**Сохранение/загрузка**:
* **💾 Сохранить** / **📂 Загрузить** — JSON-файлы на сервере (`saved_networks/`).
* **⬇ Экспорт JSON** / **⬆ Импорт JSON** — скачать/загрузить файл на локальную машину.
* В JSON сохраняется: список нейронов (со всеми параметрами и позицией), синапсы (вес + задержка), рамки-группы (с цветом и комментарием).

## Юзер-кейсы

Все примеры начинают с «чистой» сети — только дефолтные сенсоры и моторы.

### 1. Простой рефлекс «иди к еде»

**Идея**: видишь еду справа → поверни направо. Двумя синапсами:

| from | to | weight | delay |
|---|---|---|---|
| `sensor_food_right` | `motor_right` | 1.0 | 5 |
| `sensor_food_left`  | `motor_left`  | 1.0 | 5 |

Дополнительно — постоянный «толкатель» вперёд. Создай **inter-нейрон** с `noise σ = 0.05` и `leak = 0.02` (он будет шуметь сам и периодически стрелять ~5 Hz), соедини его с `motor_forward` (weight=1.0, delay=5).

Поставь несколько еды на поле (или включи спавн). Жук пошёл туда, где еды больше с той или иной стороны.

### 2. Избегание угроз

К предыдущей схеме добавь:

| from | to | weight | delay |
|---|---|---|---|
| `sensor_threat_left`  | `motor_right` | 1.5 | 3 |
| `sensor_threat_right` | `motor_left`  | 1.5 | 3 |

(Заметь — угроза слева → поворачиваешь направо, прочь от угрозы; противоположно еде.) Включи угрозы (ползунок «угроз в среде» = 3), и жук будет огибать их.

### 3. Уход от стен (лидар → поворот)

5 лидаров смотрят в веере вперёд. Подключи крайние к моторам поворота:

| from | to | weight | delay |
|---|---|---|---|
| `lidar_0` (крайний левый)  | `motor_right` | 1.0 | 3 |
| `lidar_4` (крайний правый) | `motor_left`  | 1.0 | 3 |
| `lidar_2` (центральный)    | `motor_backward` | 0.5 | 3 |

Жук тормозит при стене прямо и отворачивается от той, что ближе сбоку.

### 4. Голод как «модулятор» движения

Жук должен искать еду активнее когда голоден и отдыхать когда сыт. Создай inter-нейрон `forager` с `leak = 0.05`. Соедини:

| from | to | weight | delay |
|---|---|---|---|
| `sensor_hunger` | `forager` | 0.6 | 1 |
| `forager` | `motor_forward` | 1.0 | 5 |

Без голода `forager` молчит → жук неподвижен. По мере роста голода `sensor_hunger` стреляет всё чаще → `forager` тоже → жук двигается вперёд. Скомбинируй с пунктом 1 — получится агент, который сам начинает охоту когда проголодался.

### 5. Усталость как тормоз

`sensor_fatigue` подключи **тормозом** к `motor_forward`:

| from | to | weight | delay |
|---|---|---|---|
| `sensor_fatigue` | `motor_forward` | **−1.0** | 1 |

(Отрицательный вес = тормозный синапс, рисуется красным.) Жук побегал → устал → собственный мотор подавлен → отдых → усталость спадает → снова бежит. Получается естественный цикл активности/покоя.

### 6. Память: пара нейронов-триггеров

Два inter-нейрона, возбуждающие друг друга. Создай `mem_a` и `mem_b`:

| from | to | weight | delay |
|---|---|---|---|
| `mem_a` | `mem_b` | 1.1 | 10 |
| `mem_b` | `mem_a` | 1.1 | 10 |

Один кратковременный спайк извне (например, от сенсора угрозы) запускает цикл `mem_a → mem_b → mem_a → …` который сам себя поддерживает. Чтобы такая «память» не зацикливалась навсегда, увеличь leak (например `leak=0.15`) — тогда без подкрепления цикл затухнет за пару секунд.

Подключи `mem_*` к какому-нибудь поведению (например, к `motor_backward`) — получишь «после угрозы N секунд продолжает пятиться, даже если угрозы уже нет».

### Полезные приёмы наблюдения

* **Снижение частоты** (ползунок «частота сети» → 10 Hz) — увидишь как импульсы медленно бегут по синапсам, удобно для отладки задержек.
* **Сектора сенсоров** видны на канвасе — двигай еду вокруг агента и смотри какой именно сенсор зажёгся.
* **Группы** (⬚ Рамка) — полезно сгруппировать нейроны по функции («сенсорика угрозы», «генератор движения», «память») с цветом и комментарием — потом легче читать большую схему.
* **Шум** на inter-нейроне (`σ = 0.05`) — самый простой способ получить «спонтанный пэйсмейкер», который раз в ~1 с сам выдаёт спайк.

## Архитектура

```
app.py              — Flask + Socket.IO, фоновый поток симуляции
snn.py              — LIF-сеть на NumPy: топология, шаг, импульсы, save/load JSON
environment.py      — арена, агент, сенсорные сектора, голод/усталость, спавн объектов
static/             — фронтенд:
  environment.js    —   канвас среды (рендер, зум, инструменты)
  editor.js         —   канвас редактора (нейроны, синапсы, импульсы, мульти-выбор, рамки)
  app.js            —   socket.io клиент, верхняя панель, слайдеры
  style.css         —   стили
templates/index.html — единственная страница
saved_networks/     — серверные JSON-сохранения
```

**Темпы симуляции:**

* Симуляция: до 100 шагов/с (`SIM_DT = 10 мс`, регулируется ползунком 1–100 Hz).
* Бродкаст состояния на фронт: 30 Hz (независимо от sim_hz).
* На 300 нейронов один шаг LIF — десятки микросекунд (плотная матрица весов 300×300 в `float32`, один matvec в NumPy).
* Передача данных: ~3 КБ на снапшот при пустой схеме, ~30 КБ при ~300 активных нейронах + сотне летящих импульсов.

**Производительность фронта:**

* Один rAF-loop на канвас, без двойной отрисовки.
* Индексы активности в `Map` (без `indexOf` в горячем цикле).
* Плоская заливка нейронов вместо радиального градиента.
* Slider-эмиты дросселированы до ~10 Hz.

## Socket-протокол (если нужно интегрироваться)

Клиент → сервер:

| event | payload | действие |
|---|---|---|
| `control` | `{action: 'play'\|'pause'\|'reset_agent'\|'reset_network'}` | управление симуляцией |

| `place_object` | `{kind, x, y, [w, h]}` | поставить food/threat/obstacle/agent |
| `remove_object` | `{kind, id}` | удалить объект |
| `clear_objects` | `{kind}` | очистить категорию |
| `world_params` | `{food_target, threat_target, threat_lifetime, hunger_rate, fatigue_action_gain, fatigue_decay}` | параметры мира |

| `add_group` / `update_group` / `remove_group` | `{id, x, y, w, h, label, color, comment}` | визуальные рамки |

| event | payload | смысл |
